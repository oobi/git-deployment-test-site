# Git Deployment Test Site

A minimal PHP site used to test and validate git-based deployment pipelines. Simulates the kinds of pre/post-deploy operations you'd run on a real project.

## Stack

- **PHP** — `index.php` entry point, reads `data.txt` and renders it
- **Composer** — [`nesbot/carbon`](https://carbon.nesbot.com/) for human-readable dates
- **Node / Laravel Mix** — compiles `resources/sass/app.scss` → `public/css/app.css`

---

## First-time setup

```bash
composer install
npm install
npm run build
```

---

## Deployment commands

### Pre-deploy
```bash
# Nothing required — the deploy itself pulls updated code
```

### Post-deploy
```bash
# Install/update PHP dependencies from the updated composer.lock
composer install --no-dev --optimize-autoloader

# Rebuild assets if package.json / SASS changed
npm install --omit=dev
npm run build

# Clear the temp scratch directory
rm -rf temp/*
```

---

## Development

```bash
npm run dev     # watch + compile (unminified)
npm run build   # one-off production build (minified)
```

---

## Project structure

```
index.php              # Entry point
data.txt               # Content file rendered on the page
resources/
  sass/
    app.scss           # SASS source
public/
  css/
    app.css            # Compiled CSS (gitignored, generated by build)
temp/                  # Writable scratch dir (gitignored, .gitkeep tracks it)
vendor/                # Composer dependencies (gitignored)
node_modules/          # NPM dependencies (gitignored)
webpack.mix.js         # Laravel Mix config
composer.json
package.json
```
